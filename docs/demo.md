# Cleanroom — Demo guide

The 2-minute Loom storyline + how to run live mode if you want to push to a real Salesforce dev org.

---

## 2-minute Loom flow (the recruiter-facing path)

| Time      | Show                                                                                          |
|-----------|-----------------------------------------------------------------------------------------------|
| 0:00–0:15 | `head -15 data/seed/accounts.csv` — point at the **Acme Corp / ACME CORP / Acme, Inc.** rows. |
| 0:15–0:30 | `python scripts/run_demo.py --size 1000`. Console prints each stage timing.                   |
| 0:30–1:00 | HTML report auto-opens. Walk: top-line stats → issue table → fill-rate bars → merge groups.    |
| 1:00–1:30 | Switch to Salesforce dev org (if recorded). Show pushed records + the 4 `cleanroom_*__c` fields. |
| 1:30–2:00 | `/cleanroom data/seed/accounts.csv data/seed/contacts.csv` (slash command). Close on `README` "What this is not". |

Total demo wall-clock: ~2 min. Edit at 2x in Loom for upload.

## Run the demo (dry-run, no credentials)

```bash
git clone https://github.com/ryanmichaels-jpg/cleanroom
cd cleanroom
./scripts/setup.sh
source .venv/bin/activate
python scripts/run_demo.py --size 1000
```

Expected:
- 5 stages run in **under 5 seconds** on 1000 accounts / ~2800 contacts.
- HTML opens automatically (`reports/before_after.html`).
- Five output files land in `data/`: `audit/issues.jsonl`, `audit/merge_plan.json`, `audit/accounts_resolved.csv`, `enrichment/accounts_enriched.csv`, `push/push_manifest.jsonl`.

## Run live mode (real Apollo + real Claude)

1. Sign up for Apollo's free tier (100 credits/mo): https://app.apollo.io
2. Get an Anthropic API key: https://console.anthropic.com
3. Copy keys into `.env`:
   ```bash
   cp .env.example .env
   # then edit ANTHROPIC_API_KEY=... and APOLLO_API_KEY=...
   ```
4. Run:
   ```bash
   python scripts/run_demo.py --size 1000 --live
   ```

What `--live` changes:
- LLM tie-breaker uses **Claude Haiku 4.5** instead of the deterministic heuristic.
- Apollo provider hits the real `/v1/organizations/enrich` endpoint (cached to `data/enrichment/cache.json`).
- Claude web-search provider uses **Sonnet 4.6** to infer from domain.
- Report summary narrative is generated by Sonnet 4.6.

## Push to a real Salesforce dev org

### One-time: create the SF dev org

1. Sign up: https://developer.salesforce.com/signup
2. Once you can log in, go to **Setup → My Personal Information → Reset My Security Token** and copy the token from the email.
3. Add SF creds to `.env`:
   ```
   SF_USERNAME=you@example.com
   SF_PASSWORD=...
   SF_SECURITY_TOKEN=...
   SF_DOMAIN=login
   ```

### One-time: create the custom fields on Account

Cleanroom needs four custom fields on the Account object:

| Field name                          | Type    | Purpose                                        |
|-------------------------------------|---------|------------------------------------------------|
| `External_Id__c`                    | Text 64 (unique, External ID) | upsert key      |
| `cleanroom_audit_date__c`           | Date    | when this row was last audited                 |
| `cleanroom_confidence_score__c`     | Number 4,2 | avg confidence of enriched fields (0–1)     |
| `cleanroom_dedup_canonical_id__c`   | Text 32 | canonical id this row was merged into          |
| `cleanroom_enrichment_sources__c`   | Text 255 | pipe-delimited list of provider sources       |

The setup script creates all five via the Metadata API:

```bash
python scripts/setup_sf_schema.py
```

Or click through in **Setup → Object Manager → Account → Fields & Relationships → New**.

### Push for real

```bash
python scripts/run_demo.py --size 1000 --live --commit
```

The `--commit` flag is what flips the push from "log manifest" to "Bulk API upsert."

**Safety:** `DRY_RUN=1` in `.env` blocks the live push even with `--commit`. To actually push, set `DRY_RUN=0` (or remove the line) in your `.env`.

## Troubleshooting

**`ModuleNotFoundError: No module named 'cleanroom'`** — macOS gotcha. The `.pth` file inherited `UF_HIDDEN` from `Documents/Claude/`. Re-run `chflags -R nohidden .venv` or rerun `./scripts/setup.sh`.

**`Apollo returned 401`** — your `APOLLO_API_KEY` is wrong or expired. Apollo's free tier sometimes resets keys; re-check at https://app.apollo.io/#/settings/integrations/api.

**`Salesforce session expired`** — your security token rotated. Reset it in SF and update `.env`.

**Report shows 0 enrichment hits for Apollo** — expected in dry-run unless your dataset includes the hero domains (acme.com, globex.com, initech.com). In live mode, Apollo will hit on most real domains.

**Demo regenerates the seed every time** — pass `--regenerate-seed` to force regeneration, or delete `data/seed/accounts.csv` to trigger it.
